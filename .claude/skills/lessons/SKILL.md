# Lessons

세션을 마무리하기 전, 이번 대화에서 발견한 교훈을 정리하고 적용처를 제안한다.

## Process

### Step 1: 교훈 추출
현재 세션의 대화를 돌아보며 아래 유형의 교훈을 식별한다:
- **실패/삽질**: 시도했다가 안 된 것, 예상과 다르게 동작한 것
- **발견**: 몰랐던 사실, 새로운 패턴, 유용한 도구/API
- **결정**: 트레이드오프를 고민하고 내린 설계/구현 결정
- **개선 아이디어**: "다음엔 이렇게 하면 좋겠다"

교훈이 없으면 "이번 세션에서는 특별한 교훈이 없습니다." 출력 후 종료.

### Step 2: 적용처 매핑
각 교훈에 대해 적용 가능한 위치를 제안한다:

| 적용처 | 언제 |
|--------|------|
| `memory/` | 사용자 선호, 프로젝트 컨텍스트, 피드백 |
| `PLAN.md` | 후속 작업, 로드맵 변경 |
| `CLAUDE.md` | 반복 적용할 행동 규칙 |
| `SKILL.md` | 특정 스킬의 프로세스 개선 |
| `~/dapi-ssot/` | vault 노트 업데이트, 새 인사이트 |

### Step 3: 사용자 출력
아래 형식으로 출력한다:

```
## Lessons from this session

### 1. {교훈 제목}
{1~2문장 설명}
→ 적용처: `{파일/위치}` — {구체적으로 뭘 할지}

### 2. {교훈 제목}
...

적용할 항목 번호를 알려주세요 (예: "1,3 적용해", "전부 적용", "패스")
```

### Step 4: 적용
사용자가 선택한 항목만 반영한다. SKILL.md/CLAUDE.md 변경은 diff를 먼저 보여주고 확인 후 적용.
