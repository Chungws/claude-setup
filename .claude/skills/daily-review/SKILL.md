# Daily Review

## Trigger
사용자가 `/daily-review` 또는 자연어로 리뷰를 요청할 때 활성화.
`$ARGUMENTS`에 날짜가 있으면 해당 날짜, 없으면 오늘 날짜의 로그를 리뷰한다.

## Input
- date: 리뷰 대상 날짜 (optional, default: 오늘. 형식: YYYY-MM-DD)
- scope: all | logs-only | notes-only (optional, default: all)

## Step 1: 로그 수집
1. `~/dapi-ssot/SOT/session-logs/`에서 대상 날짜의 로그 파일들을 Glob으로 찾기
2. 로그가 없으면 "리뷰할 세션 로그가 없습니다" 출력 후 종료
3. 각 로그 파일을 읽고 기본 정보 수집:
   - skill 종류, 토픽, status
   - 수집 결과 수치
   - 오류/메모 내용
   - `status: interrupted`인 세션 식별 (hook에 의해 자동 마감된 것 = Phase 4 미완료)

## Step 2: 구조 점검 (전수 검사)
세션 로그의 "생성된 파일" 목록에서 **모든** 파일을 bash/grep으로 일괄 검사한다.
파일을 하나하나 Read할 필요 없이 grep으로 빠르게 돌린다.

### 구조 체크리스트 (bash/grep으로 전수 검사)
1. **프론트매터 필드** — `tags:`, `related:`, `status:` 존재 여부
2. **위키링크** — `[[` 가 본문에 1개 이상 있는가
3. **연결 섹션** — `## 연결` 이 파일에 있는가 (**허브 노트 면제** — 허브는 키워드맵/읽기순서가 연결 역할)
4. **허브 연결** — `topics/` 가 포함된 위키링크가 있는가 (**허브 노트 면제** — 자기 자신)
5. **플레이스홀더** — `{placeholder}`, `(목록)`, `(있으면 기록)` 등 미완성 텍스트
6. **URL 존재** — `url:` 필드가 있고 비어있지 않은가 (논문/레포/아티클만 해당)

### 내용 점검 (타입별 1개씩 샘플링)
구조 점검 통과한 파일 중 **논문 1, 레포 1, 아티클 1, 허브 1, 인사이트 1** (총 최대 5개)을 Read하여:
7. **내용 충실도** — 핵심 섹션이 3문장 이상인가

### 허브 노트 추가 체크
8. **키워드 맵** — 키워드별 분류가 있는가
9. **읽기 순서** — 추천 읽기 순서가 있는가
10. **딥다이브** — 딥다이브 추천이 있는가
11. **인사이트 섹션** — 인사이트 노트 링크가 있는가

## Step 3: 크로스링크 분석
1. 허브 노트의 `[[위키링크]]`가 실제 존재하는 파일을 가리키는지 확인
2. 끊어진 링크(dangling link) 목록 작성
3. 고아 노트(어디서도 링크되지 않은 노트) 식별

## Step 4: 수정 및 제안

### 수정 범위 규칙

**자동 수정 (vault 파일 — 데이터):**
- 프론트매터 빠진 필드 추가 (tags, related, status)
- 끊어진 위키링크 수정
- 플레이스홀더 텍스트 제거
- 고아 노트에 허브 백링크 추가

**절대 자동 수정하지 않음:**
- 내용이 부실한 노트 → 이슈로만 기록
- 관련도 낮아 보이는 노트 → 이슈로만 기록
- 중복 의심 노트 → 이슈로만 기록

**제안만 (SKILL.md / CLAUDE.md — 코드):**
- 반복되는 품질 문제 → "이런 규칙을 추가하면 좋겠다" 제안 + 구체적 diff 제시
- SKILL.md/CLAUDE.md는 **사용자 확인 없이 절대 수정하지 않는다**
- 사용자가 "고쳐" 또는 "적용해"라고 하면 그때 수정

## Step 5: 리뷰 노트 작성
`~/dapi-ssot/SOT/learnings/{date}-review.md`에 기록:

```
---
created: {date}
type: daily-review
sessions_reviewed: N
notes_checked: N
notes_sampled: N
issues_found: N
auto_fixes: N
---

# Daily Review — {date}

## 세션 요약
| 세션 | 스킬 | 토픽 | 상태 | 수집 | 이슈 |
|------|------|------|------|------|------|
| {파일명} | {skill} | {topic} | {status} | 논문N/레포N/아티클N | {이슈수} |

## 구조 점검 (전수)
전체 {N}개 검사

### 통과: {N}개
### 이슈: {N}개
- {파일명}: {어떤 항목이 fail} → {자동 수정 / 리포트만}

## 내용 점검 (샘플)
타입별 1개씩 {N}개 샘플링
- {파일명}: pass / {이슈 내용}

## 크로스링크
- 끊어진 링크: {N}개
- 고아 노트: {N}개

## 자동 수정 내역
1. {파일명}: {무엇을 수정했는지}

## 스킬 개선 제안 (사용자 확인 필요)
1. {어떤 스킬}: {제안 내용}
   ```diff
   - 기존 내용
   + 제안 내용
   ```
```

## Step 6: 사용자 출력
리뷰 노트 작성 후 **대화에 직접** 요약 출력:

```
## Daily Review: {date}

### 세션 {N}개 리뷰
- {스킬}: {토픽} — {상태} ({이슈 요약})

### 품질
구조 점검: {N}개 전수 검사 → {pass}개 통과 / {fail}개 이슈
내용 점검: {N}개 샘플 → {pass}개 통과 / {fail}개 이슈

### 자동 수정: {N}건
### 스킬 개선 제안: {N}건 (적용하려면 "고쳐"라고 해주세요)
### 리뷰 노트: {경로}
```
