# Daily Review

## Trigger
사용자가 `/daily-review` 또는 자연어로 리뷰를 요청할 때 활성화.
`$ARGUMENTS`에 날짜가 있으면 해당 날짜, 없으면 오늘 날짜의 로그를 리뷰한다.

## Input
- date: 리뷰 대상 날짜 (optional, default: 오늘. 형식: YYYY-MM-DD)
- scope: all | logs-only | notes-only (optional, default: all)

## Step 1: 로그 수집
1. `~/dapi-ssot/SOT/session-logs/`에서 대상 날짜의 로그 파일들을 Glob으로 찾기
2. 로그가 없으면 "리뷰할 세션 로그가 없습니다" 출력 후 종료
3. 각 로그 파일을 읽고 기본 정보 수집:
   - skill 종류, 토픽, status
   - 수집 결과 수치
   - 오류/메모 내용
   - `status: interrupted`인 세션 식별 (hook에 의해 자동 마감된 것 = Phase 4 미완료)

## Step 2: 노트 품질 점검
각 세션 로그의 "생성된 파일" 목록에서 **무작위 3~5개** 노트를 샘플링하여 읽고 체크:

### 체크리스트 (항목당 pass/fail)
1. **프론트매터 완성도** — tags, related, status 모두 존재하는가?
2. **위키링크** — 본문에 `[[위키링크]]`가 1개 이상 있는가?
3. **연결 섹션** — `## 연결` 섹션이 노트 끝에 있는가?
4. **허브 연결** — 허브 노트(`[[topics/...]]`)로의 링크가 있는가?
5. **플레이스홀더 잔존** — `{placeholder}`, `(목록)`, `...` 등 미완성 텍스트가 없는가?
6. **내용 충실도** — 핵심 섹션(요약, 주요 내용 등)이 3문장 이상인가?
7. **URL 유효성** — url 필드가 있고 비어있지 않은가?

### 허브 노트 추가 체크
8. **키워드 맵** — 키워드별 분류가 있는가?
9. **읽기 순서** — 추천 읽기 순서가 있는가?
10. **딥다이브** — 딥다이브 추천이 있는가?

## Step 3: 크로스링크 분석
1. 허브 노트의 `[[위키링크]]`가 실제 존재하는 파일을 가리키는지 확인 (Glob)
2. 끊어진 링크(dangling link) 목록 작성
3. 고아 노트(어디서도 링크되지 않은 노트) 식별

## Step 4: 개선사항 도출
분석 결과를 종합하여 3가지로 분류:

### 즉시 수정 (이 리뷰에서 직접 고침)
- 끊어진 위키링크 수정
- 빠진 프론트매터 필드 추가
- 플레이스홀더 텍스트 제거

### 스킬 개선 제안 (SKILL.md 수정 후보)
- 반복되는 품질 문제 → 해당 서브에이전트 스킬에 규칙 추가
- 누락 패턴 → 오케스트레이터 지시 보강

### CLAUDE.md 규칙 추가 후보
- vault 전체에 적용할 새 규칙

## Step 5: 리뷰 노트 작성
`~/dapi-ssot/SOT/learnings/{date}-review.md`에 기록:

```
---
created: {date}
type: daily-review
sessions_reviewed: N
notes_sampled: N
issues_found: N
fixes_applied: N
---

# Daily Review — {date}

## 세션 요약
| 세션 | 스킬 | 토픽 | 상태 | 수집 | 이슈 |
|------|------|------|------|------|------|
| {파일명} | {skill} | {topic} | {status} | 논문N/레포N/아티클N | {이슈수} |

## 노트 품질 점검
샘플링: {N}개 / 전체 {M}개

### 통과
- {파일명}: 모든 항목 pass

### 이슈 발견
- {파일명}: {어떤 항목이 fail} → {수정 여부}

## 크로스링크 분석
- 끊어진 링크: {N}개
- 고아 노트: {N}개
- {구체적 목록}

## 적용한 수정
1. {파일명}: {무엇을 수정했는지}

## 스킬 개선 제안
1. {어떤 스킬}: {제안 내용}

## CLAUDE.md 규칙 후보
1. {규칙 내용}
```

## Step 6: 사용자 출력
리뷰 노트 작성 후 **대화에 직접** 요약 출력:

```
## Daily Review: {date}

### 세션 {N}개 리뷰
- {스킬}: {토픽} — {상태} ({이슈 요약})

### 품질 점수
샘플 {N}개 중 {pass}개 통과 / {fail}개 이슈
주요 이슈: {가장 빈번한 문제}

### 적용한 수정: {N}건
### 스킬 개선 제안: {N}건
### 리뷰 노트: {경로}
```
